## Types of Websites

<ProjectCard
  title="Static Site"
  bg="linear-gradient(to right, #D4145A 0%, #FBB03B 100%)"
>
  The most basic type of website. It is made up of HTML pages, CSS and Javascript.This approach is good for SEO because search engines can crawl the site and get full rendered HTML pages back with content.
</ProjectCard>

<ProjectCard
  title="Single Page Application (SPA)"
  bg="linear-gradient(to right, #662D8C 0%, #ED1E79 100%)"
>
  Only requires a single initial server request for a blank HTML page to be sent back to the browser. Making SPAs faster than traditional static websites. Additionally, SPAs are easier to update. 
</ProjectCard>

<ProjectCard
  title="Server Side Rendered (SSR)"
  bg="linear-gradient(to right, #009245 0%, #FCEE21 100%)"
>
  HTML pages are built on the fly on the server. This approach is good for SEO because the server sends back a content rich HTML page for each request. It is also easier to update than static websites. 
</ProjectCard>

<ProjectCard
  title="Static Website Generator (Gatsby)"
  bg="linear-gradient(to right, #D585FF 0%, #00FFEE 100%)"
>
  Final HTML pages are compiled at build time. They are good for SEO & are also easy to update. Overall, SSGs offer a good balance between SEO, speed and ease of update. 
</ProjectCard>
